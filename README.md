# helpdesk-hub

ヘルプデスク・アプリケーションサポート向けの問い合わせ管理システムです。

## プロダクトの目的

`helpdesk-hub` は、単なる問い合わせ受付フォームではなく、運用現場で必要な以下の流れを一気通貫で扱うことを目的にしています。

- 問い合わせの受付
- 状況管理（ステータス遷移）
- 担当者アサイン
- エスカレーション
- SLA 管理
- FAQ 候補化
- 対応品質の可視化

## 想定ユーザー

- 一次受付担当
- 二次対応担当
- 管理者
- 依頼者

## 想定利用シーン

- 社内ヘルプデスク
- 情シス窓口
- アプリケーションサポート窓口
- 小規模チームの問い合わせ運用

## 開発方針

実装は以下の3フェーズで段階的に進めます。

1. **MVP**: 認証、問い合わせ CRUD、ステータス更新、コメント、検索
2. **実務拡張**: SLA、エスカレーション、履歴、権限制御
3. **アピール拡張**: ダッシュボード、FAQ候補化、レポート/CSV、テスト拡充

詳細な要件と実装タスクは `docs/requirements.md` と `docs/issue-backlog.md` を参照してください。

## ドキュメント運用（4バージョン統合）

本リポジトリでは、以下4つの観点を整合させて運用します。

- Version 1: `README.md`（プロダクト定義）
- Version 2: `docs/requirements.md`（要件定義）
- Version 3: `docs/issue-backlog.md`（実装タスク）
- Version 4: `docs/version-integration.md`（整合ルール）

統合レビュー手順と差分調停ルールは `docs/version-integration.md` を参照してください。
