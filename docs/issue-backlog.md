# Issue Backlog (Prioritized)

転職用ポートフォリオとして、実装順・依存関係・成果物が明確になるように 30 件に分解しています。

## P0: フェーズ1（MVP）

### 1. [P0] プロジェクト初期化と技術基盤
- **目的**: Next.js + TypeScript + Tailwind + Prisma + PostgreSQL 構成を作る
- **完了条件**:
  - ローカルでアプリ起動できる
  - DB 接続確認できる
  - CI 用 lint/test スクリプトが定義されている

### 2. [P0] 認証基盤（Auth.js / NextAuth）
- **目的**: ログイン/ログアウトとセッション管理
- **完了条件**:
  - ログイン・ログアウトできる
  - 未認証時に保護ページへアクセス不可

### 3. [P0] ロール定義と表示制御の骨組み
- **目的**: requester / agent / admin ロールを導入
- **完了条件**:
  - 画面単位のアクセス制御が機能
  - UI 表示分岐がロールで切り替わる

### 4. [P0] Prisma スキーマ初版作成
- **目的**: users / tickets / categories / comments / histories を定義
- **完了条件**:
  - migration 実行できる
  - seed で最小データ投入できる

### 5. [P0] 問い合わせ登録 API とフォーム
- **目的**: ticket 作成を実装
- **完了条件**:
  - title/description/category/priority を登録可能
  - Zod バリデーションエラーを表示

### 6. [P0] 問い合わせ一覧 API + 画面
- **目的**: 基本一覧表示を実装
- **完了条件**:
  - テーブルで tickets 一覧を表示
  - ページング（または件数制限）あり

### 7. [P0] 問い合わせ詳細 API + 画面
- **目的**: 1 件の詳細情報表示
- **完了条件**:
  - 基本情報・担当・期限・履歴概要が見える

### 8. [P0] ステータス更新 API
- **目的**: ステータス変更を可能にする
- **完了条件**:
  - 許可遷移のみ更新可能
  - 不正遷移で API がエラーを返す

### 9. [P0] ステータス遷移ルールの型安全化
- **目的**: union/enum + ルールマップでドメイン制約を明文化
- **完了条件**:
  - フロント/サーバで同一の遷移定義を利用
  - 遷移テストが追加されている

### 10. [P0] 担当者アサイン機能
- **目的**: assignee を設定/変更可能にする
- **完了条件**:
  - 一覧/詳細から担当変更できる
  - 変更履歴が記録される

### 11. [P0] コメント機能
- **目的**: 対応コメント追加
- **完了条件**:
  - 時系列でコメント表示
  - 投稿者・投稿日時を記録

### 12. [P0] 変更履歴表示（ステータス/担当）
- **目的**: 変更トレーサビリティを担保
- **完了条件**:
  - チケット詳細で履歴一覧を確認できる

### 13. [P0] 一覧検索・絞り込み
- **目的**: 実運用で必要な絞り込み
- **完了条件**:
  - キーワード/ステータス/カテゴリ/優先度/担当者に対応

### 14. [P0] 共通エラーハンドリング/トースト通知
- **目的**: UX と運用性向上
- **完了条件**:
  - API エラー時に利用者へ明確なメッセージ表示

## P1: フェーズ2（実務拡張）

### 15. [P1] SLA フィールド追加（期限の保持）
- **目的**: first_response_due_at / resolution_due_at 管理
- **完了条件**:
  - ticket 作成/更新時に期限を保持できる

### 16. [P1] SLA 期限計算ロジック
- **目的**: 優先度・カテゴリ起点で期限を算出
- **完了条件**:
  - ルールに基づいた期限自動設定
  - テストで算出結果を検証

### 17. [P1] SLA 超過/間近の可視化
- **目的**: 対応漏れ防止
- **完了条件**:
  - 一覧で色分け/バッジ表示される

### 18. [P1] エスカレーション機能
- **目的**: 一次→二次対応の明示的切替
- **完了条件**:
  - 理由・日時を保存
  - ステータスが Escalated に遷移可能

### 19. [P1] 添付ファイル機能（MVP版）
- **目的**: スクリーンショット/ログ添付
- **完了条件**:
  - ファイル upload と一覧表示
  - 許可拡張子/サイズ検証あり

### 20. [P1] 通知機能（アサイン/期限/更新）
- **目的**: 重要イベントを見逃さない
- **完了条件**:
  - 通知テーブル記録
  - 画面上で未読/既読管理可能

### 21. [P1] FAQ 候補化フラグ
- **目的**: 解決済みチケットからナレッジ化導線を作る
- **完了条件**:
  - Resolved/Closed チケットを FAQ 候補登録可能

### 22. [P1] 権限制御の本実装
- **目的**: requester は自分のチケットのみ閲覧
- **完了条件**:
  - API レベルでアクセス制御
  - E2E で権限境界を検証

### 26. [P2] 監査ログ詳細

### 29. [P2] テスト拡充（Vitest + Playwright）
